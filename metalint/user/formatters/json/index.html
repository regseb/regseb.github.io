Formateurs · json

Description

Ce formateur retourne les données brutes de Metalint. Le résultat est un objet JSON dont les clés sont les fichiers analysés et les valeurs : la liste des problèmes remontés. Si un fichier respecte un des patrons globaux, mais qu'aucun checker ne lui est associé : la valeur sera null.

Une option est disponible : indent, pour choisir la taille des indentations :

  • 0 (par défaut) : le fichier n'aura pas de retours à la ligne ;
  • 1 : le document aura des retours à ligne et une indentation d'une espace.
  • 2 : une indentation de deux espaces ;
  • etc.

Exemple


{
    "reporters": {
        "formatter": "json",
        "options": { "indent": 4 }
    }
}

Voici un rapport (avec l'indentation à 4) où trois fichiers sont présents : .gitignore qui ne respecte aucun patron des checkers, index.html qui n'a aucune erreur et script.js qui contient deux erreurs (une remontée par JSHint et l'autre par JSCS). 


{
    ".gitignore": null,
    "index.html": [],
    "script.js": [
        {
            "linter": "jshint",
            "rule": "W083",
            "severity": 2,
            "message": "Don't make functions within a loop.",
            "locations": [
                {
                    "line": 255,
                    "column": 36
                }
            ]
        },
        {
            "linter": "jscs",
            "rule": null,
            "severity": 2,
            "message": "Comments cannot contain the following keywords: todo, fixme",
            "locations": [
                {
                    "line": 278,
                    "column": 3
                }
            ]
        }
    ]
}
