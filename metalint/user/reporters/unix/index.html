unix (Rapporteur)

Description

Ce rapporteur affiche les données de façon compacte : une ligne égale une notification.

  • file : le nom du fichier ;
  • line : le numéro de la ligne ;
  • column : le numérode la colonne ;
  • message : le message décrivant le problème ;
  • linter (si la verbosité est supérieure ou égale 1) : le linter ayant trouvé le problème ;
  • rule (si la verbosité est supérieure ou égale à 2) : la règle qui n'est pas respectée.

Les éléments sont séparée par deux-points. Sauf pour le linter et la rule qui sont affichés entre parenthèse et séparés par un point : (linter.rule).

Une ligne vide est insérée entre les notifications des fichiers différents.

Exemple

Voici un rapport (avec la verbosité à 2) où trois alertes sont affichés. 


README.md:1:: First header should be a h1 header (markdownlint.MD002)

tool.js:55:36: Don't make functions within a loop (jshint.W083)
tool.js:78:3: Comments cannot contain the following keywords: todo, fixme (jscs)
