json (Rapporteur)

Description

Ce rapporteur retourne les données brutes de Metalint. Le résultat est un objet JSON dont les clés sont les fichiers vérifiés et les valeurs : la liste des problèmes remontés. Si un fichier respecte un des patrons globaux, mais qu'aucun checker ne lui est associé : la valeur ser null.

En passant le niveau de verbosité à 1 : la sortie JSON est indentée.

Exemple

Voici un rapport (avec la verbosité à 1) où trois fichiers sont présents : .gitignore qui ne respecte aucun patron des checkers, global.js qui n'a aucune erreur et script.js qui contient deux erreurs (une remontée par JSHint et l'autre par JSCS). 


{
    ".gitignore": null,
    "global.js": [],
    "script.js": [
        {
            "linter": "jshint",
            "rule": "W083",
            "severity": 2,
            "message": "Don't make functions within a loop.",
            "locations": [
                {
                    "line": 255,
                    "column": 36
                }
            ]
        },
        {
            "linter": "jscs",
            "rule": null,
            "severity": 2,
            "message": "Comments cannot contain the following keywords: todo, fixme",
            "locations": [
                {
                    "line": 278,
                    "column": 3
                }
            ]
        }
    ]
}
